Authentication

Create a new account

Create a new user account with the desired username and password for authentication. Also associate a name with their login information.

Request Body schema: application/json
name
string [ 2 .. 256 ] characters
username
string [ 2 .. 64 ] characters
password
string >= 8 characters

Responses

The default address for the API

http://localhost:5000/v1/authentication/register

Request samples

  • Payload
Content type
application/json
{
  • "name": "Alex Krantz",
  • "username": "akrantz01",
  • "password": "testing-password"
}

Response samples

  • 400
  • 409
Content type
application/json
{
  • "code": 400,
  • "message": "missing required fields"
}

Create a new todo

Create a new todo for a user with some title. Optionally, it can also have a longer description, a category, some tags, and a due date.

Authorizations:
cookieAuth
Request Body schema: application/json
title
string [ 1 .. 128 ] characters

A short title for the todo

content
string

Extra information about the todo

due
string <date-time>

The date and time when the todo is due

category
integer

The ID of the category to associate with

tags
Array of integers

Tags to add to the todo

Responses

The default address for the API

http://localhost:5000/v1/todos

Request samples

  • Payload
Content type
application/json
{
  • "title": "Assignment 3",
  • "content": "Some assignment for school",
  • "category": 4,
  • "tags": [
    • 9,
    • 3
    ],
  • "due": "2021-08-25T23:59:59.000Z"
}

Response samples

  • 201
  • 400
  • 401
Content type
application/json
{
  • "id": 3
